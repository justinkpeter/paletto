# Paletto

Image-driven color palette tool for designers and developers. Upload any photo, extract its colors, and get AI-generated names for each one. Reorder swatches, lock favorites across regenerations, save palettes to your account, and share them via URL.

Live at: [paletto.netlify.app](https://paletto.netlify.app)

---

## Features

- **Color extraction** — upload any image and extract a 6-color palette using Color Thief
- **AI color naming** — each color gets an evocative, design-forward name generated by Claude
- **Drag to reorder** — rearrange swatches with live visual feedback powered by dnd-kit
- **Lock colors** — lock individual swatches so they survive palette regeneration
- **Spacebar to regenerate** — instantly generate a new palette variation from the same image
- **Export** — copy all hex codes to clipboard in one click
- **Save palettes** — authenticated users can save palettes with the inspiration image attached
- **Shareable URLs** — every saved palette gets a public `/palette/[id]` page
- **Inline renaming** — click any palette name to rename it in place
- **Dark mode** — full light/dark mode with persistent preference

## Tech Stack

| Layer | Choice | Why |
|---|---|---|
| Framework | Next.js 15 (App Router) | Server components, API routes, and file-based routing in one package |
| Language | TypeScript | Type safety across the full stack |
| Styling | SCSS Modules | Design token system with Japandi-inspired warm aesthetic |
| Color extraction | Color Thief | Client-side palette extraction, no server round-trip needed |
| AI naming | Anthropic Claude (claude-sonnet-4-5) | Contextual, evocative color names that understand palette relationships |
| Auth | Supabase Auth (Google OAuth) | Passwordless auth with zero backend setup |
| Database | Supabase Postgres | Row-level security ensures users only access their own data |
| Storage | Supabase Storage | Inspiration images stored per-user with UUID-based paths |
| Drag and drop | dnd-kit | Accessible, performant sortable with live position feedback |
| Deployment | Netlify | Zero-config deployment with environment variable management |

## Architecture Decisions

**Client-side color extraction** — Color Thief runs entirely in the browser on the uploaded image. No image is sent to a server for processing, which keeps extraction instant and avoids unnecessary API costs.

**AI naming via API route** — The Claude API call runs server-side in a Next.js API route, keeping the API key out of the client bundle. All 6 colors are sent in a single request for contextual naming — Claude sees the full palette and can distinguish similar tones.

**Stable drag IDs** — Each color entry gets a `crypto.randomUUID()` on creation. Using stable IDs instead of array indices prevents dnd-kit from losing track of items during reorder animations.

**Per-swatch loading state** — Instead of a single global `isLoading` flag, each `ColorEntry` has its own `isLoading` boolean. This means locked colors stay fully rendered while unlocked ones show individual shimmer skeletons during regeneration.

**Row-level security** — Every Supabase table has RLS policies enabled. Users can only read and write their own palettes and collections, enforced at the database level rather than just the application layer.

## Local Development

```bash
# Install dependencies
npm install

# Set up environment variables
cp .env.example .env.local
# Fill in NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY, ANTHROPIC_API_KEY

# Run dev server
npm run dev
```

---

## Environment Variables

| Variable | Description |
|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | Your Supabase project URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Your Supabase anon/public key |
| `ANTHROPIC_API_KEY` | Your Anthropic API key for color naming |
